# Team Agent Orchestrator

## Team Composition

| Role | Agent File | Specialty |
|------|-----------|-----------|
| UI Designer | `agents/ui-designer.md` | Design, UX, Stitch prototyping |
| Product Manager | `agents/pm.md` | Requirements, Jira, prioritization |
| Full Stack Dev | `agents/fullstack-dev.md` | Code implementation, testing |
| QA Tester | `agents/qa-tester.md` | Testing, bug verification, quality assurance |

## How to Use

### Invoke a single agent
Ask me to "talk to the [role]" or "ask the [role] to [task]".

### Run a team discussion
Ask me to "run a team meeting about [topic]" — I'll spawn all three agents and synthesize their perspectives.

### Workflow Example
1. **PM** defines requirements and user stories
2. **UI Designer** creates mockups and design specs
3. **Full Stack Dev** implements the feature
4. **PM** reviews against acceptance criteria
5. **UI Designer** reviews for design fidelity

## Team Collaboration Rules
- PM has final say on scope and priority
- Designer has final say on UX and visual decisions
- Developer has final say on technical approach
- All agents should flag risks and trade-offs to each other
